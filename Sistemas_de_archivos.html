Sistemas de Archivos

Estructura de almacenamiento y organización de datos en Kali Linux

Un sistema de archivos es el sistema de almacenamiento de un dispositivo de memoria, que estructura y organiza la escritura, búsqueda, lectura, almacenamiento, edición y eliminación de archivos de una manera concreta. Los sistemas de archivos también otorgan a los archivos, entre otras, las siguientes características:

  • Convenciones para nombrar a los archivos
  • Atributos de archivo
  • Control(es) de acceso
ext4

ext4 admite sistemas de archivos más grandes y maneja más archivos que ext3, lo que lo hace adecuado para las necesidades de almacenamiento modernas.

  • Es el sistema de archivos predeterminado en Kali Linux.
  • Es robusto, rápido y confiable.
  • Soporta archivos grandes y volúmenes grandes.
  • Buen manejo de fallos.
FAT32 / exFAT

Establecido en 1996, FAT32 es uno de los viejos rockeros del mundo de los sistemas de archivo, robusto pero anticuado. Su mayor limitación es que sólo permite guardar archivos de hasta 4 GB. exFAT fue introducido por Microsoft como una actualización para superar esta limitación.

  • Usados mayormente para compatibilidad con otros sistemas operativos (Windows, macOS).
  • Útiles para unidades USB o discos externos.
  • exFAT permite archivos de más de 4GB, a diferencia de FAT32.
NTFS

Se trata de otra alternativa al sistema FAT32 promovida por Microsoft, además es el sistema de archivos que Windows utiliza por defecto. Sin los límites del tamaño máximo de archivo del FAT32, el NTFS se convierte en una muy buena opción para discos duros y otras unidades externas.

  • Sistema de archivos nativo de Windows.
  • Kali puede leer y escribir en NTFS gracias al soporte de ntfs-3g.
  • También útil para compartir datos entre sistemas.
Btrfs

Fue diseñado con el objetivo de implementar características avanzadas, al mismo tiempo que se enfoca en la tolerancia a fallos, la reparación y la facilidad de administración.

  • Más moderno, con características avanzadas como snapshots, compresión, y verificación de datos.
  • No es el predeterminado en Kali, pero está disponible.
  • Ofrece mejor protección contra la corrupción de datos.
